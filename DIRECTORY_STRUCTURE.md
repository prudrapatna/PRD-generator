# Chief of Staff - Project Directory Structure

## Root
*   `README.md`: Project overview and setup instructions.
*   `DIRECTORY_STRUCTURE.md`: This file.
*   `feature-context.md`: High-level feature descriptions.
*   `visual_gen_cuj.py`: Python script for generating CUJ visualizations.

## /agents
*   `prd_orchestrator_agent.md`: The "Staff PM" agent configuration, linking context and skills for PRD generation.

## /knowledge (Strategic Context & Research)
*   `strategy/`: High-level business strategy documents (`strategy_details.md`).
*   `user_research/`: Raw research, personas (`proactive_adopter_profile.md`), and synthesis (`hypertension_problem_definition.md`).
*   `target_performance/`: Benchmark data (`apple_hypertension_validation.pdf`) and competitive analysis (`competitor_analysis.md`).
*   `algorithm_overview_and_target_performance/`: Technical details on the sensing algorithms (`algorithm_overview.md`).
*   `Additional Context.md`: Definitions, tone guidelines, and sample context.

## /product_specs
*   `prds/`: Generated Product Requirement Documents (e.g., `PRD-Tidal-2026-02-16.md`).

## /skills (Agent Capabilities)
*   `product-narrative-skill.md`: Guidance for writing "7-Star" narratives and Press Releases.
*   `cuj-generator-skill.md`: Framework for defining Critical User Journeys.
*   `acceptance-criteria-skill.md`: Standards for writing engineering-ready requirements.
*   `visualization-skill.md`: Instructions for generating diagrams.
*   `presentation-skill.md`: Guide for creating slide decks.
*   `internal-comms-skill.md`: Templates for internal communication.

## /conductor (Process & Standards)
*   `tracks/`: Implementation plans and track records.
*   `code_styleguides/`: Coding standards.
*   `product-guidelines.md`: Product principles.
*   `workflow.md`: Standard operating procedures.

## /FDA_wellness_guidelines
*   `general_wellness_guideline.md`: FDA guidance on "General Wellness" vs. "Medical Device."
